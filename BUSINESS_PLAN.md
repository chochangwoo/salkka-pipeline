## 2. 문제 정의

### 내집마련 실수요자의 3가지 고통

| 고통 | 현실 | 결과 |
|------|------|------|
| **정보 과부하** | 호갱노노, 부동산 카페, 유튜브, 뉴스 — 매일 쏟아지는 정보 | 뭘 봐야 할지 모르겠다 |
| **해석 불가** | 실거래가 표, 전세가율, DSR — 숫자만 보여주고 의미를 안 알려줌 | 이게 비싼 건지 싼 건지 모르겠다 |
| **타이밍 불안** | "지금 사야 해?" "더 빠지면?" — 물어볼 사람 없음 | 결국 결정을 못 한 채 1~2년 흘려보냄 |

### 핵심 인사이트

> 사람들은 **더 많은 데이터**가 아니라 **"그래서 어쩌라고?"** 를 원한다.

살까말까는 데이터를 수집하는 서비스가 아니라, **데이터를 해석해서 의미를 전달하는 서비스**다.

## 3. 타겟 고객

### 페르소나

**김지은 (34세, 대기업 마케터, 맞벌이)**

- 연 소득 7,500만원 (맞벌이 합산 1.3억)
- 현재 마포구 전세 거주 (전세 만기 1년 남음)
- 내집마련 희망하지만 구체적 행동 못 하는 중
- 부동산 유튜브 보다가 포기, 호갱노노 가끔 검색
- "누가 정리해서 알려줬으면 좋겠다"

### 타겟 세그먼트

| 세그먼트 | 규모 추정 | 특징 |
|----------|----------|------|
| 1차: 마포구 내집마련 준비생 | ~5,000명 | 전세 거주, 30~40대, 1~2년 내 매수 의향 |
| 2차: 서울 서북권 (마포·서대문·은평) | ~15,000명 | 인접 지역 확장 |
| 3차: 서울 전체 실수요자 | ~100,000명 | 중장기 목표 |

### 왜 마포구부터?

1. **실수요 밀집**: 직주근접(여의도·마포·공덕), 젊은 맞벌이 비율 높음
2. **가격대 적정**: 강남 대비 진입 장벽 낮아 "살까 말까" 고민이 가장 활발
3. **거래 활성**: 주간 거래량 충분해 매주 데이터 기반 분석 가능
4. **확장 용이**: 서대문·용산으로 자연스럽게 확장 가능한 지역

### 유료 전환 트리거

무료 4주간 다음 경험을 제공해 "이거 없으면 불안하다"를 만든다:

1. **1주차**: 시장 온도계 + 타이밍 신호 → "오, 이런 게 있구나"
2. **2주차**: 관심 단지 임장 서술 → "직접 안 가도 느낌이 온다"
3. **3주차**: 급매 알림 미리보기(blur 처리) → "궁금한데 안 보여준다"
4. **4주차**: "무료 체험이 종료됩니다" + 지난 4주 요약 리포트

### 핵심 원칙

### 채널별 전략

#### 1단계: 시드 구독자 확보 (0~50명, 1~3개월)

| 채널 | 방법 | 목표 |
|------|------|------|
| **지인 네트워크** | 카톡/인스타로 "내가 만든 부동산 뉴스레터인데 피드백 좀" | 10~20명 |
| **부동산 카페** | 네이버 부동산 카페에 주간 분석 요약 글 + 구독 링크 | 20~30명 |
| **블라인드** | "부동산 초보" 게시판에 유용 정보 공유 + 구독 유도 | 5~10명 |

#### 2단계: 자연 성장 (50~500명, 3~12개월)

| 채널 | 방법 | 목표 |
|------|------|------|
| **구독자 추천** | "친구에게 공유" 버튼 + 공유 시 과거 호 열람 | 월 10~20명 |
| **SEO 블로그** | 매주 뉴스레터 아카이브 웹 공개 (검색 유입) | 월 5~10명 |
| **인스타그램** | 뉴스레터 핵심 내용을 카드뉴스로 변환 | 월 10~15명 |

#### 3단계: 가속 성장 (500명+, 12개월~)

| 채널 | 방법 | 목표 |
|------|------|------|
| **뉴스레터 크로스 프로모션** | 다른 뉴스레터와 상호 소개 (머니레터, 어피티 등) | 월 30~50명 |
| **소액 광고** | 인스타그램 광고 월 3~5만원 테스트 | ROI 기반 |
| **부동산 커뮤니티 제휴** | 아파트 입주민 카페, 맘카페 등 | 지역 특화 |

## 8. 운영 계획

### 주간 루틴

```
[일요일 저녁 또는 월요일 새벽]

1. main.py 상단 데이터 업데이트 (30분)
   - THIS_WEEK_NEWS: 이번 주 부동산 뉴스 1건 선정
   - THIS_WEEK_INDICATORS: 타이밍 인디케이터 5개 업데이트
   - THIS_WEEK_QNA: 유료 구독자 질문 2건 선정 (유료 전환 후)

2. git push → GitHub Actions 자동 실행

3. 월요일 오전: 발송 결과 확인 (15분)
   - 오류 체크
   - HTML 렌더링 확인
```

### 월간 루틴

```
[매월 첫째 주]
- MONTHLY_COMPARE_COMPLEXES 업데이트 (단지 비교 대상)
- 전월 KPI 리뷰 (구독자 수, 오픈율, 해지율)
- 프롬프트 품질 점검 (GPT 출력 만족도)

[매월 말]
- 다음 달 뉴스 트렌드 사전 조사
- 구독자 피드백 정리
```

## 9. 기술 우위

### 왜 쉽게 따라하기 어려운가

| 요소 | 진입 장벽 |
|------|----------|
| **파이프라인 자동화** | 수집→분석→리포트→발송 전 과정 코드 완성. 처음부터 만들려면 2~4주 |
| **프롬프트 엔지니어링** | 4종 프롬프트 최적화 완료. 톤·분량·형식 조절에 수십 회 반복 필요 |
| **비용 구조** | GPT-4o/mini 혼합으로 주당 ~500원. 경쟁자가 4o만 쓰면 5~10배 비용 |
| **구독자 데이터** | 시간이 갈수록 쌓이는 오픈율·클릭률·전환 데이터 = 콘텐츠 최적화 자산 |
| **지역 전문성** | 마포구 집중 운영으로 쌓이는 지역 특화 인사이트 |

### 기술 스택 비용 효율

```
전체 기술 스택 월 비용: ~1,700원

vs 일반적인 뉴스레터 비용:
- Mailchimp (500명):  ~15,000원/월
- 서버 (AWS/Vercel):  ~10,000원/월
- DB (RDS/Planet):    ~15,000원/월
- 합계:              ~40,000원/월

→ 살까말까는 경쟁 대비 96% 비용 절감
```

| 경쟁자 출현 | 중 | 낮음 | 선점 효과 + 구독자 관계 + 지역 전문성으로 차별화 유지 |
| 본업과 시간 충돌 | 중 | 중간 | 주당 1.5시간 이내 운영. 자동화 비율 극대화 |

### 핵심 재무 특성

| 특성 | 내용 |
|------|------|
